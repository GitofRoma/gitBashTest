1)Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout [file-name]

2)Как вернуть файл из индекса в рабочий каталог?
git restore --staged [file name]  

3)Как отменить ошибочный commit ?
git reset HEAD ~ [number]

4)Как создать новую ветку и переключится в нее?
git checkout -b [branch name]

5)В чем отличение между fast-forward merge  and non fast-forfard merge?
fast-forward merge может быть выполнен очень быстро и не приводит к созданию новых коммитов, т.к. в главной ветке не было изменений, а в случае non fast-forfard merge есть другие изменения в главной ветке и Git создает новый коммит, который представляет объединение двух веток.

6)Когда может возникнуть конфликт слияния веток, как  его разрешить?
Конфликт слияния веток может возникнуть, когда две ветки, которые вы пытаетесь объединить, содержат изменения в одних и тех же строках кода или файлов, но эти изменения взаимно несовместимы. Решение - вручную выбрать, какие изменения вы хотите сохранить.

7)Можно ли удалить ветку? Как? 
перейти на другую ветку
git branch -d [branch name]
